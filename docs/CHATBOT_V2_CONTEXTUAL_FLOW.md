# Chatbot V2 - Contextual Flow Engine

تمت إضافة محرك سياق للمساعد بحيث لا يتعامل مع كل رسالة كسؤال مستقل فقط، بل يحفظ آخر نية وخطوة داخل جلسة المحادثة.

## أهم التحسينات

- حفظ `last_intent`, `current_intent`, `current_step`, `context_data` داخل `chat_sessions`.
- فهم المتابعات مثل: `البريد مكتوب صحيح` و `هل ممكن الخطوات بشكل صحيح` بناءً على آخر مشكلة.
- حماية فحص البريد الإلكتروني: لا يتم كشف هل البريد موجود أم لا داخل الدردشة.
- إضافة `actions` في رد API لعرض أزرار عملية مثل فتح تسجيل الدخول، استعادة كلمة المرور، صفحة التواصل، البحث والصفوف.
- إرجاع `step` مع كل رد لتسهيل تتبع مسار المحادثة.
- تحويل الردود إلى خطوات مرقمة عند الحاجة بدل الرد العام.

## مثال API جديد

```json
{
  "session_id": 1,
  "answer": "خطوات حل مشكلة عدم وصول رسالة التفعيل...",
  "intent": "email_verification_problem",
  "step": "verification_steps",
  "actions": [
    {"label":"فتح صفحة تسجيل الدخول","type":"link","url":"/login","style":"primary"},
    {"label":"فتح صفحة التواصل","type":"link","url":"/contact-us","style":"secondary"}
  ],
  "suggestions": ["البريد مكتوب صحيح", "فحصت البريد غير الهام", "ما زالت المشكلة موجودة"]
}
```

## ملاحظة تشغيل

لا تحتاج إلى SQL يدوي للجداول الجديدة في الوضع الطبيعي؛ `AutoMigrate` يضيف الأعمدة الجديدة عند تشغيل الباك اند إذا كان مستخدم قاعدة البيانات يملك صلاحيات ALTER.
